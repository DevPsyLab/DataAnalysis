Mplus is software for structural equation modeling.

4 Variables

4.1 Specify Variables

VARIABLE:
    NAMES = ID age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID age x1 x2 y1;

4.2 Categorical Variables

VARIABLE:
    NAMES = ID age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID age x1 x2 y1;
    CATEGORICAL = x1 x2;

4.3 Count Variables

VARIABLE:
    NAMES = ID age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID age x1 x2 y1;
    COUNT = x1 x2;

4.4 Cluster Variable

There are multiple ways of accounting for nested data in structural equation modeling. One way to account for nested data is to use multilevel structural equation modeling. Another approach is to use a cluster variable to generate cluster-robust standard errors of parameters. To use a cluster variable, specify CLUSTER under the VARIABLE section, and specify TYPE = COMPLEX under the ANALYSIS section:

VARIABLE:
    NAMES = ID age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID age x1 x2 y1;
    CLUSTER = ID;

ANALYSIS:
    TYPE = COMPLEX;

4.5 Individually Varying Times of Observation

VARIABLE:
    NAMES = ID age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID age x1 x2 y1;
    TSCORES = age;

4.6 Auxiliary Variables

VARIABLE:
    NAMES = ID age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID x1 x2 y1;
    AUXILIARY = age;

4.7 Sampling Weight Variable

VARIABLE:
    NAMES = ID wt age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID age x1 x2 y1;
    WEIGHT = wt;

4.8 Multilevel Variables

Between- and within-cluster variables:

VARIABLE:
    NAMES = ID wt age x1 x2 x3 y1;
    MISSING = .;
    USEVARIABLES = ID age x1 x2 y1;
    WITHIN = x1;
    BETWEEN = x2;

7 Model

BY

short for measured by – defines latent variables

example: f1 BY y1-y5;

ON

short for regressed on – defines regression relationships example: f1 ON x1-x9;

PON

short for regressed on – defines paired regression relationships

example: f2 f3 PON f1 f2;

WITH

short for correlated with – defines correlational relationships

7.1 Define Latent Variables

MODEL:
    y1 BY x1 x2 x3;

7.2 Regression Paths

Regress outcome variable on predictor variable(s):

MODEL:
    y1 ON x1 x2;

7.3 Covariance Paths

MODEL:
    x1 WITH x2;

7.4 Indirect Effects

MODEL INDIRECT:
    y IND x;

7.5 Parameter Label

To specify a parameter label, provide the label in parentheses after the parameter:

MODEL:
    y1 BY x1* x2 x3 (load1-3);
    y2 BY x4* x5 (load5) x6;

9 Setting Parameter Constraints

9.1 Freeing a Parameter

By default, the first loading on a factor is fixed to zero. You can freely estimate the parameter by adding an asterisk:

MODEL:
    y1 BY x1* x2 x3;

9.2 Setting Two Parameters to be Equal

To set two parameters to be equal, provide the same parameter label for each parameter.

9.3 Setting Lower and Upper Bounds on a Parameter

To set lower and upper bounds on a parameter, you can assign the parameter a parameter label. Then, you can assign the constraint to the parameter (via the label) under the MODEL CONSTRAINT section. For example, to constrain a parameter between 0–1,

MODEL:
    y1 BY x1* x2 x3 (load3);

MODEL CONSTRAINT:
    load3 > 0; load3 < 1;
